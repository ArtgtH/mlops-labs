## Бизнес-цели проекта
1) Минимизировать пропуски мошенничества (FN)
2) Не создавать много ложных тревог (FP)
3) Модель должна быть повторяемо обучаемой, метрики - измеряемыми на отложенной выборке, а пайплайн — стабильным. В конце курса проект должен быть готов к внедрению)))

## Метрики
TP - (модель=fraud; real=fraud)
TN - (модель=не fraud; real=не fraud)
FP - (модель=fraud; real=не fraud) 
FN - (модель=не fraud; real=fraud)

1) Recall для fraud = TP / (TP + FN)
2) F1-score для fraud = 2TP / (2TP + FP + FN)
